## 七日省察

以下三份文件构成当前可实施基线：

1. [最终产品与安全规格](specs/2026-07-13-seven-day-reflection-research-design.md)
2. [工程实施计划与验收标准](plans/2026-07-13-seven-day-reflection-chat.md)
3. [已确认的墨香风格视觉稿](../seven-day-reflection-preview.html)

发生冲突时，按“最终规格 → 实施计划 → 视觉稿”的顺序判断业务行为；视觉布局与用户可见文案以视觉稿和 `AGENTS.md` 中的稳定文案约束为准。

七日省察的最终关键决策：回顾昨天向前七个完整自然日；每天成功生成一次且不可重生成；一段总结加最多一个“另一种可能”；最多两个尝试计划；完整聊天保存在本机；只有长期记忆跨线程；页面标题为“七日省察”，底部导航为“省察”。

其余带日期的 `specs/` 与 `plans/` 文件记录对应功能的设计过程。修改已确认行为时，应同时更新相关规格、实施计划、视觉稿、测试和 `AGENTS.md`。
