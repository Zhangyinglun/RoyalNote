# 起居注首页视觉改造实施方案

**状态：** 视觉方向已确认，待实施
**视觉基准：** [`docs/royalnote-design-review.html`](../../royalnote-design-review.html)
**目标平台：** Android / Jetpack Compose / Material 3
**改造范围：** 主页、主页速录编辑器、记录时间线、底部导航的共用视觉

## 1. 目标

把已确认的 HTML 视觉方向落实到 Android 首页：保留宣纸、深墨、铜色和衬线字，减少重复卡片边框，让速录成为页面唯一主焦点，并把记录列表改成开放册页式时间线。

本次是纯 UI 改造。`ViewModel`、Room、Repository、记录排序、日期分组、导入、时间编辑、保存、修订和删除行为不变。

## 2. 决策优先级

出现冲突时按以下顺序判断：

1. `AGENTS.md` 中的稳定业务规则与精确文案；
2. 本实施方案中的交互与适配规则；
3. HTML 视觉稿中的构图、层级、色彩和装饰细节。

HTML 是视觉基准，不作为新增业务规则的依据。

## 3. 不在本次范围内

- 不增加搜索、心情筛选、云同步、导出或批量清理。
- 不改变心情枚举、记录字段或数据库结构。
- 不改变时间选择器、保存校验和删除确认的结果语义。
- 不把首页视觉样式直接套到“省察”和“设置”的内容区；底部导航作为共用组件会同步调整。
- 不引入图片背景、纹理资源、第三方 UI 库或新字体文件。

## 4. 已确认的视觉结构

### 4.1 页面骨架

首页保持 `Scaffold + LazyColumn`，由上至下为：

1. 系统状态栏与应用标题栏；
2. 引言“录今日之事，存此刻之心。”及铜色短线/圆点；
3. 速录编辑器；
4. 日期标题；
5. 开放式时间线记录；
6. 系统导航栏之上的底部导航。

页面背景直接使用主题 `background`，不添加模拟纸纹。纸张层次通过背景、编辑器表面、细边框和极轻阴影完成。

### 4.2 标题栏

- “起居注”使用 `displaySmall`，目标为 32sp、Serif、SemiBold，字距约 0.08em。
- “导入”保留文字按钮和至少 48dp 的触控区域，不新增图标。
- 标题栏底部使用低对比 1dp 分隔线。
- 继续使用真实 Android 状态栏和 edge-to-edge，不绘制 HTML 中的模拟时间、电量或信号图标。

### 4.3 引言

- 页面左右常规留白 16dp；窄屏可降到 12dp。
- 引言使用 16sp Serif 和放松字距，颜色为 `onSurfaceVariant`。
- 下方保留 40dp 铜色短线与 3dp 金色圆点。
- 引言与速录卡之间只保留轻微间距，使二者形成一个连续开场区。

### 4.4 速录编辑器

速录编辑器是首页唯一完整容器：

- 8dp 圆角，1dp 外边框，内部再绘制一圈低透明度细线；
- 左侧上部使用 3dp × 44dp 铜色强调条；
- 表面颜色比页面背景略亮，阴影保持很轻；
- 标题“速录一则”下方使用 18dp 铜色短线；
- “入录”是此屏唯一实色主按钮，宽度铺满，视觉高度 44dp，实际触控高度不低于 48dp；
- 编辑态继续显示“修订此则”“改毕入录”和“作罢”，但沿用同一容器和视觉层级。

事件输入框保留 `OutlinedTextField` 的文本、光标、错误和无障碍能力，但弱化默认 Material 容器感：

- 标签仍为“今日何为”；
- 空值保存仍显示“先写下做了什么”；
- 最小两行，允许内容自然增高；
- 不固定速录卡高度，避免大字体或心绪补述出现裁切；
- 显示非限制性的当前字数，例如“0 字”，不实施 HTML 中演示用的 200 字上限。

### 4.5 时间选择

视觉上合并成一个轻量时间区，但保留开始/结束日期和时间四个可操作入口：

- 同日且宽度充足时，一行显示“始 今天 08:30 — 终 今天 09:30”；
- 跨日、窄屏或字体放大时自动换成开始/结束两行；
- 日期和时间仍分别打开现有 `DatePicker` 与 `TimePicker`；
- 继续保留 `startDateButton`、`startTimeButton`、`endDateButton`、`endTimeButton` 测试标签和语义说明；
- 使用 24dp 线性时钟图标及低对比底部分隔线，不使用新的实色按钮。

### 4.6 心绪选择

七种心绪保持一行展示，顺序和文案完全不变：

`开心`、`满足`、`平静`、`疲惫`、`烦躁`、`低落`、`焦虑`。

每项由 48dp 可点击单元、34–36dp 圆形图标框和 12sp 标签组成。未选中时使用细轮廓；选中时使用铜色双线感边框、铜色文字和底部小圆点。点击已选项仍取消选择。

窄屏适配使用 `BoxWithConstraints`：

- 宽度大于 375dp 时，内容水平留白 16dp；
- 360–375dp 时，内容水平留白 12dp；
- 七个 48dp 触控单元用 `Arrangement.SpaceBetween` 排列，图标视觉尺寸可以缩小，但点击目标不缩小；
- 标签允许正常显示，不以横向滚动或第二行替代已确认的一行构图。

新增七枚单色 Android VectorDrawable。图标只表达情绪区别，颜色仍由现有 `moodColor()` 提供；文字标签和选中语义承担主要可访问性信息。

### 4.7 开放式时间线

移除每条记录的完整 Card、内层边框和左强调条，改为三列结构：

| 列 | 建议宽度 | 内容 |
|---|---:|---|
| 时间 | 48dp | 开始/结束时间，使用等宽数字感的 12sp 标签样式 |
| 轨道 | 24dp | 1dp 连接线和 9dp 铜色节点 |
| 正文 | 自适应 | 事件文本、心绪补述、心绪标记、记录操作 |

记录之间用低对比 1dp 横线分隔，最后一条不显示横线。时间线轨道连续贯穿同一日期组，并保留末端空心节点。

当前数据模型只有一个 `eventText`，因此不会像 HTML 示例那样自动拆成伪标题和正文。实际记录使用一个 `bodyLarge` 文本块完整展示；`moodNote` 单独以 `bodyMedium` 展示。

记录心绪不再使用完整填充的 `AssistChip`，改为“小图标 + 心绪文字”的紧凑标记；颜色继续使用七种已定义心绪色。

### 4.8 修订和删除

HTML 为突出阅读节奏没有展示记录操作，但 Android 必须保留现有能力。每条记录正文右上方增加 48dp 的“更多”按钮，打开 `DropdownMenu`：

- `修订`：进入现有行内编辑器；
- `抹去`：继续弹出“抹去此则？”确认对话框。

菜单按钮默认使用 `onSurfaceVariant`，不占用主色，不在每条记录下方重复展示两个文字按钮。

### 4.9 底部导航

- 保留“主页”“省察”“设置”和现有图标资源；
- 移除 Material 默认胶囊选中底色；
- 选中项使用铜色图标/文字，并在标签下方显示 24dp × 2dp 铜色短线；
- 未选中项使用 `onSurfaceVariant`；
- 组件背景使用 `surface`，顶部有 1dp 分隔线；
- 每项触控区域不低于 48dp；
- 使用真实 `navigationBarsPadding()`，不绘制 HTML 中的模拟手势横条。

### 4.10 深浅色

继续跟随系统深浅色，`dynamicColor = false`。不因为 HTML 默认展示浅色而移除深色模式。

只使用现有主题色：

- 浅色：`#F2EDE3`、`#EAE4D6`、`#2A2015`、`#63584A`、`#7A5C3E`、`#D6CDBB`；
- 深色：`#1A1814`、`#262320`、`#EDE6D6`、`#B8AE9A`、`#C49A6C`、`#3D3833`；
- 心绪继续使用 `Color.kt` 中已经确认的七种颜色。

## 5. 文件改动规划

| 文件 | 计划改动 | 业务风险 |
|---|---|---:|
| `ui/theme/Type.kt` | 增补 `displaySmall`、`headlineSmall` 等首页所需 Serif 字阶 | 低 |
| `MainActivity.kt` | 重构标题区、速录容器、心绪选择、日期标题和记录时间线 | 中 |
| `ui/TimeRangeFields.kt` | 改为可换行的轻量时间区，保留现有选择器和测试标签 | 中 |
| `ui/AppNavigation.kt` | 自定义克制型底部导航选中状态和系统 inset | 中 |
| `res/drawable/ic_mood_*.xml` | 新增七枚单色心绪图标 | 低 |
| `res/drawable/ic_clock_24.xml` | 新增线性时钟图标 | 低 |
| `res/drawable/ic_more_vert_24.xml` | 新增记录更多菜单图标 | 低 |
| `androidTest/.../RoyalNoteAppTest.kt` | 按新结构补充选择、菜单和时间入口测试；稳定文案断言不删 | 中 |

为降低回归风险，本轮不移动 `RoyalNoteApp`、`RecordEditor`、`RecordCard` 的包名或公开签名。可在 `MainActivity.kt` 内先完成视觉重构；只有当单文件明显失控时，再把纯展示组件提取到 `ui/HomeVisualComponents.kt`，不移动状态和事件处理。

## 6. 实施顺序

### 阶段一：令牌、图标和预览基线

- [ ] 截图留作后续阶段对比基线。

### 阶段二：标题和速录编辑器

- [ ] 调整 TopAppBar、引言、装饰线和页面间距。

- [ ] 调整事件输入框、错误提示和非限制性字数提示。

- [ ] 以紧凑心绪标记替换填充 Chip。

### 阶段四：底部导航和质量验证

- [ ] 更新 UI 测试并运行构建、单测、Lint。
- [ ] 在实体机或模拟器上截图，对照 HTML 检查首页首屏构图。

## 7. 必须保留的行为与文案

- 空事件不能保存，提示“先写下做了什么”。
- 编辑记录保留 `createdAt`，只更新 `updatedAt`。
- 记录顺序仍为 `startedAt DESC, createdAt DESC, id DESC`。
- 日期仍显示“今天”“昨天”或 ISO 日期。
- 心绪标签和“未输入”语义不改。
- “导入”“修订”“抹去”“作罢”“入录”“改毕入录”等现有行为不改。
- 导航仍为“主页”“省察”“设置”，嵌套导入路由不变。
- UI 测试依赖的中文文本节点必须继续存在；必要时只调整节点层级和测试查找方式。

## 8. 验收标准

### 视觉

- 430 × 900dp、默认字体大小、浅色模式下，首页首屏可看到完整速录编辑器、日期标题和至少一条记录的主要内容。
- 标题、引言、速录、时间线和底部导航的层级与 HTML 视觉基准一致。
- 除速录编辑器外，记录列表没有重复的整块卡片边框。
- 页面每个视区只有一个实色主按钮。
- 深色模式不出现纯黑大块、亮白边框或高饱和装饰色。

### 响应式与无障碍

- 360 × 800dp 下无水平溢出，七种心绪仍为一行，点击目标至少 48dp。
- 字体放大到 1.5 倍时无文字裁切；时间区和按钮允许换行或增高。
- 所有图标按钮具有中文 `contentDescription`；装饰图标不重复朗读。
- 心绪项暴露选中状态，更多菜单和删除确认可由 TalkBack 完成。
- 颜色不作为心绪选择和导航选中的唯一提示。

### 功能与工程

- 新增、编辑、取消编辑、删除确认、心绪选择/取消、日期选择、时间选择、导入导航全部通过。
- JVM 单元测试、Android UI 测试和 Lint 不因视觉改造新增失败。
- 不新增依赖，不修改数据库 schema，不修改 ViewModel/Repository 接口。

## 10. 主要风险与控制

| 风险 | 控制方式 |
|---|---|
| 七个心绪项在窄屏拥挤 | 48dp 固定触控单元 + 窄屏 12dp 页边距 + 缩小视觉圆框，不缩点击区域 |
| 时间字段在跨日或大字体下溢出 | 同日一行、跨日/窄屏自动两行；保留四个独立语义入口 |
| 开放时间线弱化编辑/删除可发现性 | 每条记录保留 48dp 更多菜单，菜单项使用现有准确文案 |
| HTML 示例有标题/正文，但数据只有一个事件字段 | 不做自动拆句，完整展示 `eventText` |
| HTML 的 `0/200` 被误解为业务限制 | Android 只显示当前字数，不增加长度上限 |
| 首屏高度与无障碍冲突 | 不锁死组件高度；默认字号追求构图一致，大字体优先完整可读和可滚动 |
| 视觉改造影响现有精确文本测试 | 先保留所有稳定文本节点，再更新结构相关查找和补充语义测试 |

## 11. 完成定义

只有同时满足以下条件，首页视觉改造才算完成：

1. Compose 首页达到已确认 HTML 的层级与气质；
2. 深色、窄屏和 1.5 倍字体通过人工检查；
3. 所有原有记录行为和稳定文案保持；
4. 构建、单测、Lint 以及可运行环境下的 UI 测试通过；
5. 用户基于 Android 实机或模拟器截图完成最终视觉确认。
