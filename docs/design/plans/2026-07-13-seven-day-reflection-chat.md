# 七日省察与反思伙伴实施计划

**状态：** 产品与视觉确认完成，可直接实施。最终规格以 [`../specs/2026-07-13-seven-day-reflection-research-design.md`](../specs/2026-07-13-seven-day-reflection-research-design.md) 和 [`../../seven-day-reflection-preview.html`](../../seven-day-reflection-preview.html) 为准。

**目标：** 将空白“分析”模块替换为可用的“七日省察”：每日固定七日回顾、本机聊天历史、受控长期记忆、微型行动实验与危机提示。

**日期口径：** 用户本地日期当天首次打开时，回顾昨天往前七个完整自然日。生成成功后固定，不重新生成；生成时保存记录快照。

**技术方案：** 继续使用单 Activity、Navigation Compose 2、Material 3、Room、StateFlow、OkHttp 与 kotlinx.serialization。聊天 UI 使用 Compose `LazyColumn`、`OutlinedTextField`、WindowInsets/IME API，不引入第三方即时通讯 SDK。模型回复采用一次非流式结构化请求。

## 已确认产品边界

- 页面标题为 `七日省察`，底部导航为 `省察`。
- 回顾只读取七日记录快照和当前长期记忆，不读取聊天。
- 零记录时生成本地空回顾，不调用模型；聊天仍可使用，但不得推断近期状态。
- 每日线程和完整消息保存在 Room；旧线程可继续发送。
- 新线程不读取其他线程原文，跨线程只使用当前长期记忆。
- API 上下文使用固定回顾、对应记录快照、当前记忆、滚动对话摘要和近期消息；完整历史不删除。
- 明确目标、已接受行动和执行反馈可自动收录；模式、压力来源和自我认识必须确认；诊断、人格标签和危机内容拒绝写入。
- 危机场景暂停普通分析，显示通用安全提示，不写入长期记忆。
- 不实现重新生成、导出、批量清空或用户可见的证据详情。

## 数据结构

1. `daily_reflections`
   - 线程日期、七日区间、结构化回顾 JSON、记录快照 JSON、Prompt 版本、创建/更新时间。
2. `reflection_messages`
   - 消息 ID、线程日期、角色、正文、发送状态、安全提示标记、创建时间。
3. `memory_candidates`
   - 来源消息、分类、内容、确认策略、处理状态、创建时间。
4. `reflection_threads`
   - 线程日期、滚动摘要、已摘要到的消息 ID、更新时间。
5. `filesDir/reflection/MEMORY.md`
   - 稳定 ID 的精简长期记忆；应用内以分组卡片管理。

Room 从版本 2 升至 3，使用显式迁移新增表并保留现有起居记录。

## 业务组件

- `ReflectionContextBuilder`：计算七个完整自然日、查询记录并生成不可变快照。
- `OpenRouterReflectionService`：生成一段总结、最多一个“另一种可能”和最多两个尝试计划，并生成聊天回复、安全状态、记忆候选与滚动摘要。
- `ConversationContextBudget`：按保守字符预算选择近期消息和待摘要片段；完整消息始终保留在 Room。
- `MemoryPolicy`：本地重新判断自动、待确认和拒绝规则。
- `ReflectionRepository`：持久化回顾、线程、消息和候选；管理 `MEMORY.md`。
- `ReflectionViewModel`：管理今日生成、历史线程、聊天、重试、行动和记忆。

## Prompt 与上下文

- `seven_day_reflection_v1.md`：每日回顾资源；逻辑版本保存为 `seven_day_reflection_v2`。
- `reflection_chat_v1.md`：非流式聊天回复、安全状态与记忆候选。
- `memory_review_v1.md`：滚动对话摘要，不负责直接改写长期记忆。
- 生成请求只发送七日记录快照和当前长期记忆，不发送聊天。
- 聊天请求发送固定回顾、对应记录快照、当前长期记忆、滚动摘要与近期消息。
- 近期消息预算为 18,000 字符，超过 22,000 字符触发摘要；摘要失败时仍使用有界近期窗口，避免请求因上下文过长而失败。
- OpenRouter 使用设置页当前模型与 reasoning effort；回复采用结构化 JSON，不启用流式传输。

## UI

- 默认线程页：日期区间、一张只含一段总结与最多一个“另一种可能”的七日回望卡、最多两张尝试计划卡、聊天消息与固定输入栏。
- 顶栏入口：`往日`、`记忆`。
- 历史页：按日期倒序显示本地线程，支持打开并继续聊天，顶栏与系统返回键回到当前省察。
- 记忆页：按栏目显示卡片，支持修订、删除和处理待确认候选，顶栏与系统返回键回到当前省察。
- 墨香样式：8dp 卡片、青铜左边线、宣纸背景、用户青瓷气泡、Serif 字体、低饱和状态色。

## 错误与安全

- 今日回顾失败：保留最近成功回顾作为带日期的只读缓存，并提供重试；只有成功保存才算今日已生成。
- 聊天失败：用户消息保留为“未送达”，原地重试且不重复插入。
- 缺少 API Key：显示前往设置提示；历史与缓存仍可离线查看。
- 明确即时危险：显示固定安全回复，允许继续输入，不产生普通分析、实验或记忆候选。
- 模型返回未知证据、非法记忆类别、过长内容或禁止内容时，本地丢弃。

## 验证

- JVM：日期窗口、零/少量记录、证据过滤、Prompt 注入边界、上下文压缩、消息重试、记忆策略、Markdown 存储与 ViewModel 状态。
- Compose：标题/底栏文案、回顾卡片、聊天输入和消息、历史导航、记忆候选、键盘避让与空/错误状态。
- Gradle：`testDebugUnitTest`、`lintDebug`、`assembleDebug`；有设备时运行 `connectedDebugAndroidTest`。

## 验收标准

- 首次进入当天省察时，日期范围准确落在昨天向前七个完整自然日；当天记录不进入回顾。
- 同一天成功生成后不再调用模型；失败后可以重试，且不会把失败计为成功。
- 七日回望只显示一段总结，以及证据足够时最多一个“另一种可能”。
- 只有一至两条记录时不显示“另一种可能”或尝试计划；零记录时不调用回顾模型。
- 尝试计划最多两张，可接受、修订、略过，并可从长期记忆终止。
- 往日线程可以查看并继续聊天；往日和记忆页的顶栏返回与系统返回键均有效。
- 完整聊天保存在本机；发送失败的用户消息显示“未送达”并可原地重试。
- 新线程不读取旧聊天原文；只有已确认长期记忆跨线程使用。
- 危机提示暂停普通分析、使用固定通用安全文案、不生成记忆候选，并允许用户继续输入。
- 视觉实现遵循“墨香”设计系统，并与已确认 HTML 视觉稿保持一致。
