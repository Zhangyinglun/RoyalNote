# 原地修订记录设计

## 背景

当前点击记录卡片里的「修订」后，记录内容会进入页面顶部的「速录一则」模块。用户期望修改发生在当前记录位置：点哪条记录，就让哪条记录展开成修改模块，保存后仍回到这条记录的位置。

## 目标

- 点击某条记录的「修订」后，该记录卡片原地切换为编辑模块。
- 顶部「速录一则」只负责新增记录，不再承载已有记录的编辑。
- 保存修改后更新本地记录，并让该位置恢复为普通记录卡片。
- 取消修改后放弃本次表单内容，并让该位置恢复为普通记录卡片。

## 非目标

- 不新增页面导航、弹窗编辑、搜索、筛选、手动时间修改或云同步。
- 不改 Room 表结构和记录排序规则。
- 不改变情绪标签文案、空状态文案和删除确认流程。

## 推荐方案

采用原地替换方案。时间线渲染每条记录时，检查 `uiState.editingRecord?.id` 是否等于当前记录 `id`。如果相等，渲染编辑模块；否则渲染普通记录卡片。

顶部新增模块不再读取编辑状态。新增表单和编辑表单使用两份独立输入状态：顶部保存新增草稿，原地编辑模块保存当前记录的修改草稿。这样用户编辑旧记录时，不会被带回顶部，也不会让顶部输入框被旧记录内容覆盖。

## 状态与数据流

- `RecordTimelineViewModel` 分别保存新增表单状态和当前编辑表单状态。
- 点击「修订」调用 `startEditing(record)`，把该记录内容写入编辑表单状态，不改动顶部新增草稿。
- 时间线根据 `editingRecord.id` 决定哪张卡片进入编辑态。
- 点击顶部「入录」只新增记录；点击原地「改毕入录」只更新当前编辑记录，并保留原 `createdAt`。
- 新增保存成功后只清空顶部新增表单；编辑保存成功后只清空编辑状态，时间线重新显示更新后的记录卡片。
- 点击「作罢」调用现有 `cancelEditing()`，放弃当前编辑状态。

## UI 规则

- 编辑模块复用现有 `RecordEditor` 的视觉语言：纸面卡片、左侧铜色竖线、情绪 chip、按钮行。
- 原地编辑模块标题显示「修订此则」，按钮显示「改毕入录」和「作罢」。
- 顶部模块标题固定为「速录一则」，按钮固定为「入录」。
- 编辑过程中，顶部新增模块仍可显示用户原有新增草稿；本次改动不要求同时新增和编辑，但不能让两处表单共享同一份输入。
- 当前只允许同时编辑一条记录。点另一条记录的「修订」时，编辑态切换到另一条记录。

## 错误处理

- 编辑时正文为空仍不保存，并沿用现有提示「未记今日之事，不宜入录」。
- 更新失败沿用现有提示「入录未成，烦请再试」。
- 删除正在编辑的记录时，沿用现有行为：删除成功后清空编辑状态。

## 测试

- ViewModel 测试：进入编辑态不会覆盖顶部新增草稿。
- ViewModel 测试：编辑保存后更新记录，保留 `createdAt`，只清空编辑状态。
- Compose UI 测试：当 `editingRecord` 指向时间线里的记录时，时间线位置显示「修订此则」和「改毕入录」，顶部仍显示「速录一则」和「入录」。
- 回归测试：新增记录流程仍能显示顶部输入模块，普通记录卡片仍显示「修订」和「抹去」。
