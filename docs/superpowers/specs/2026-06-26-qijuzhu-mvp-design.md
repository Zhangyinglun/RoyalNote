# 起居注 MVP 设计

日期：2026-06-26

## 背景

「起居注」是一个本机 Android 笔记应用，用来快速记录某个时刻做了什么，以及当时的心情。第一版只做最小可用范围：新增记录、时间线展示、编辑记录、删除记录，并把数据保存到本机 SQLite 数据库中。

当前项目是一个新的 Android Compose 模板工程，只有 `MainActivity` 和基础主题文件，还没有数据库层或业务页面。

## 第一版范围

第一版支持：

- 新增记录
- 按时间线展示记录
- 编辑已有记录
- 删除已有记录
- 记录持久保存到本机 SQLite

第一版不做：

- 搜索
- 按心情筛选
- 当前状态记录
- 手动修改记录时间
- 云同步
- 导出
- 多页面导航

## 数据结构

使用 Room 管理本机 SQLite。第一版只建一张记录表。

记录字段：

| 字段 | 类型 | 规则 |
|---|---|---|
| `id` | `Long` | 自增主键 |
| `eventText` | `String` | 必填，记录“做了什么” |
| `moodTag` | `String?` | 可空；未选择时界面显示“未输入” |
| `moodNote` | `String?` | 可空，记录心情补充 |
| `createdAt` | `Long` | 自动生成，保存创建时间 |
| `updatedAt` | `Long` | 新增时等于 `createdAt`，编辑时更新为当前时间 |

心情标签固定为：

- 开心
- 满足
- 平静
- 疲惫
- 烦躁
- 低落
- 焦虑

编辑记录时保留原始 `createdAt`，只更新 `updatedAt`。时间线按 `createdAt` 倒序展示。

## 首页交互

应用名称显示为「起居注」。首页分为两块：

- 上半部分：快速记录区
- 下半部分：时间线

快速记录区包含：

- “做了什么”输入框，必填
- 心情标签单选区，可不选
- “心情补充”输入框，可空
- 保存按钮

新增记录规则：

- 只填“做了什么”也能保存
- 未选择心情时，数据库里 `moodTag` 为空，时间线显示“未输入”
- 保存成功后清空输入框，退出编辑状态，时间线立刻刷新
- “做了什么”为空或只有空白时不保存，并在界面上提示用户填写内容

## 时间线展示

时间线按日期分组，最近记录在最上面。

日期标题规则：

- 当天记录显示“今天”
- 前一天记录显示“昨天”
- 更早记录显示日期，例如 `2026-06-24`

每条记录展示：

- 创建时间，例如 `23:18`
- “做了什么”正文
- 心情标签；未选择时显示“未输入”
- 心情补充；为空时不展示这一行

## 编辑和删除

编辑规则：

- 点一条记录进入编辑状态
- 快速记录区回填该记录的“做了什么”、心情标签、心情补充
- 保存编辑后更新原记录，不新建记录
- 编辑保存不改变原来的 `createdAt`
- 编辑时提供取消操作，取消后放弃修改，回到新增状态

删除规则：

- 记录卡片上提供删除按钮
- 删除前弹出确认框
- 用户确认后删除数据库里的记录，时间线立刻刷新

## 架构

第一版采用单 Activity、单页面 Compose 架构。

实现分层：

- UI：Compose 页面和组件，负责输入、展示和用户操作
- ViewModel：维护输入状态、编辑状态、时间线状态，调用仓库方法
- Repository：封装记录的新增、查询、编辑、删除
- Room：提供 DAO、Entity、Database，落到本机 SQLite

数据流：

1. Room DAO 暴露按 `createdAt` 倒序排列的记录流
2. Repository 转发 DAO 数据
3. ViewModel 收集记录流，并把记录按日期分组给 UI 使用
4. UI 操作调用 ViewModel 方法
5. ViewModel 写入 Repository 后，Room 数据变更自动刷新时间线

## 错误处理

第一版只处理本地常见错误：

- 空正文：不保存，在界面提示“先写下做了什么”
- 删除确认：避免误删
- 数据库操作异常：保留当前输入，不清空表单，并提示保存失败或删除失败

不做复杂错误恢复，也不做崩溃日志上传。

## 验收标准

实现完成后至少验证：

- 新增记录：只填“做了什么”可以保存
- 心情：可以选择开心、满足、平静、疲惫、烦躁、低落、焦虑，也可以不选
- 时间线：记录按日期分组，最近记录在最上面
- 编辑：修改正文、心情标签、心情补充后保存，原创建时间不变
- 取消编辑：不改动原记录
- 删除：删除前有确认，确认后记录从时间线消失
- 持久化：重启应用后记录仍在
- 构建：`./gradlew assembleDebug` 通过
