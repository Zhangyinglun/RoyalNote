# 起居注 · 古风文案设计

## 原则

**有古意而不晦涩，像史官手笔而非文言堆砌。** 未来所有新增功能中的用户可见文本，均按此原则生成。

### 具体约束

- 用词偏书面、典雅，但现代人能一眼看懂
- 单字或双字为主，控制长度（UI 空间有限）
- 避免生僻字和冷僻典故
- 可参考《浮生六记》《陶庵梦忆》等明清笔记文风——浅近文言，娓娓道来
- 日期保留公历格式，不引入农历

### 替换词库参考

| 白话 | 古风 | 备注 |
|------|------|------|
| 做 / 干 | 为 | 今日何为 |
| 写 / 记录 | 录 / 记 | 入录、速录 |
| 保存 | 入录 | 收入起居注 |
| 编辑 / 修改 | 修订 | 修而订之 |
| 删除 | 抹去 | 如删削文稿 |
| 取消 | 作罢 | 作罢即可 |
| 心情 / 情绪 | 心绪 | 更具书卷气 |
| 今天 | 今日 | — |
| 昨天 | 昨日 | — |
| 补充 | 补述 | 意到即可 |
| 失败 | 未成 | 稍柔 |
| 请再试 | 烦请再试 | 带一点执礼 |
| 还没有 | 尚无 | 尚无起居之录 |
| 不能 | 不可 | 不可复还 |

## 实施方式

直接替换各源文件中的硬编码字符串，不新建常量文件（改动最小，当前项目规模无需额外抽象层）。

涉及文件：
- `MainActivity.kt` — UI 文案
- `RecordTimelineViewModel.kt` — Snackbar 消息和分组标签
- `SeedData.kt` — 种子数据精简为一条
